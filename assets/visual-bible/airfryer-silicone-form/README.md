# Visual Bible: airfryer-silicone-form

Единый источник визуальной правды для всех кампаний товара
(Ozon арт. 1931921872). Используется агентами image-producer, visual-director,
sequence-director и animation-qa (см. `VISUAL_AUTOMATION_PIPELINE.md` в корне репо).

## Файлы

- `visual_bible.json` — канон: форма (ровно 2 овальные ручки, тёмно-серый матовый
  квадрат ~18.5×18.5×5 см, рифлёное дно), аэрогриль, руки, кухня, свет, камера,
  правила еды, запрещённые изменения.
- `continuity_rules.json` — hard-fail коды, scoring (10 измерений, пороги 70/80),
  sequence-проверки, лимиты генерации, гейты.
- `references/` — указатели на референсные изображения (см. `sources.json` в каждой
  категории). Бинарники НЕ дублируются в репо — референсы живут в
  `D:/OzonGrowthProject/content/assets/`.
- `approved/` — сюда складываются утверждённые кадры кампаний
  (`<campaign_code>/<scene_id>.png` + метаданные), пока пусто.

## Чего не хватает (запросить у владельца)

Референсы ниже НЕ существуют — не придумывать, ждать от владельца:

1. **Фото реального аэрогриля владельца** — фронтально с окном + с выдвинутой
   корзиной, дневной свет. Сейчас канон прибора держится на AI-кадре видео №2
   (`place.png`) — этого мало для строгого reference match.
2. **Фото корзины изнутри** — чистой И с жиром после готовки (для сцен 1–2
   кампании coating-protect: губка, мытьё).
3. **Фото губки и раковины** — сцены 1–2 вводят новый реквизит, референса нет.
4. **Фото рук** (тыльная сторона + захват формы за ручки) — если владелец хочет
   соответствие СВОИМ рукам; иначе канон остаётся по h1/b3a видео №2.
5. **Фото блюда кампании** (3 куриных бёдрышка + дольки картофеля в форме) —
   опционально; без него еда фиксируется первым утверждённым сгенерированным кадром.

## Правило обновления

Канон меняется только через владельца. Новые утверждённые референсы добавлять в
`references/<категория>/sources.json` с пометкой origin: owner_photo | approved_generation.
